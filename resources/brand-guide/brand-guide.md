# HILAB Brand Guideline — Slide Summary

## Brand Information

- **Name**: HILAB (pronounced: /haɪ lab/)
- **Full name**: HILAB TECHNOLOGY
- **Positioning**: Data-First to AI-Powered
- **Tagline**: Make Data Work Smarter / We make impossible possible
- **Description**: Technology operations partner, helping businesses digitally transform in a practical, effective, and sustainable way
- **Domain**: Consulting and implementing digital transformation, Data, and AI solutions for enterprises

## Core Values (4 HI)

1. **HI CONNECTION** — Open in collaboration
2. **HIGH PERFORMANCE** — High performance
3. **HI HUMAN-CENTERED** — Human-centered
4. **HI HANDS-ON** — Hands-on in execution

## Brand Personality

- **Flexible**: Ready to adapt methods to each context
- **Reliable**: Accurate, consistent, professional
- **Long-term thinking**: Prioritize sustainable foundations, lasting value
- **Practical**: Feasible solutions, data-driven, avoid flashiness

## Visual Identity (Brand Guideline p.29-50)

### Visual Language
- **Wave/data flow**: Primary element, represents data flow — use soft wave curves
- **Dot matrix/dot grid**: Evokes data network — bright red dots on dark backgrounds, or light gray on light backgrounds
- **Red-black gradient**: Creates depth, technology feel — soft blends with natural glow areas

### Slide Background System — 3 main types

#### Light backgrounds (use ~70% of slides)
- White/light gray, optionally with faint white wave patterns or subtle dot waves
- Suitable for: presentations, proposals, information-dense documents
- **Variants**: (1) Clean white, (2) White + white wave curves, (3) White + light gray dot matrix, (4) White + gentle gradient to Soft Blush
- See `images/bg-light/`: 7.png (wave tunnel), 8.png (dot wave), 9.png (fabric swoosh), 10.png (minimal lines), 11.png (curved waves), 12.png (waves right), 13.png (angular waves)

#### Dark backgrounds (use ~20-25% of slides)
- Carbon Black (#1E1D1B) with red gradient (Fire Red/Ruby Red) — soft glow effect
- May include red data wave/dot matrix patterns
- Suitable for: cover, section divider, conclusions, key strategic visuals
- **Variants**: (1) Red data wave mesh, (2) Red dot matrix wave, (3) Dot particles bokeh, (4) Soft mesh wave, (5) Geometric vertical columns, (6) Horizontal dot wave band
- See `images/bg-dark/`: 1.png (wave mesh), 2.png (dot matrix), 3.png (particles), 4.png (soft mesh), 5.png (columns), 6.png (horizontal dots)

#### Red accent backgrounds (use sparingly ~5-10%)
- Full red gradient with glow areas at corners
- Only for strong CTAs, special highlights, critical moments
- Use `bg-dark/3.png` (dense red particles) or `bg-dark/5.png` (red geometric columns) for equivalent high-impact effect

### Image Style
- **Abstract images**: Use red-white-black tones, representing data flow, technology, connectivity
- **Photography**: Prioritize lighting, modern feel, youthful and dynamic spirit. Modern workspaces, digital devices, team interactions
- **DO**: Natural, authentic, proactive imagery. Modern workspace, people interacting with technology
- **AVOID** (p.44 Brand Guideline): Generic blue stock images (globe networks, holograms), people touching virtual screens, city skylines with neon, IT hexagon/diagram stock. All are outdated and inconsistent with HILAB brand

### Infographics (p.46-47)
- Use multi-shade red palette (Fire Red → Ruby Red → Soft Blush)
- Chevron arrows, pie charts, donut charts, pyramids, flow diagrams — all use red palette
- Bar charts: Series 1 Fire Red, Series 2 Ruby Red or lighter
- Line charts: Fire Red line, red dot markers
- Pie/Donut: Gradient from Fire Red → Ruby Red → Soft Blush → neutral
- Total red area should not exceed 30-40% of chart surface area

### Extended Chart Color Palette (p.36)
When needing more than 3-4 series in charts, use extended palette:
- Row 1 (warm): #FFF572, #FEBB3B, #F26A36, #7DCE1, #0BCF2, #171B60, #BCB6A6, #621B46, #006B35
- Row 2 (cool/neutral): #9CBCCD, #3D5567, #22303A, #F7B3CC, #E871AB, #840055, #47BF4F, #006B35
- Row 3 (earth): #CCB3D7, #8E5BA6, #361460, #D0B2A4, #72635D, #453F3D
- Brand color (red) always serves as the primary highlight

## Logo

### Usage Rules
- **Primary logo**: HILAB (text), used in most cases
  - White: `images/logo/hilab-white.png` → dark/red backgrounds
  - Black: `images/logo/hilab-black.png` → light backgrounds
  - Red: `images/logo/hilab-red.png` → light backgrounds, brand emphasis
- **Secondary logo**: HI (compact), only when space is limited
  - White: `images/logo/hi-icon-white.png` → dark/red backgrounds
  - Black: `images/logo/hi-icon-black.png` → light backgrounds
  - Red: `images/logo/hi-icon-red.png` → light backgrounds, accent
- Safe spacing: 1/2 of logo height
- NEVER: gradients, drop shadows, outlines, scale distortion, diagonal placement
